# PR Review Toolkit

プルリクエストの包括的なレビューを実行します。

## 引数

$ARGUMENTS - PR番号またはブランチ名（省略時は現在のブランチ）

## 実行内容

### 1. 変更概要の取得

```bash
# PR番号が指定された場合
gh pr view $ARGUMENTS --json title,body,files,additions,deletions

# ブランチの場合
git log main..HEAD --oneline
git diff main...HEAD --stat
```

### 2. レビューチェックリスト

以下の観点で詳細にレビュー:

#### 機能面
- [ ] 要件を満たしているか
- [ ] エッジケースが考慮されているか
- [ ] エラーハンドリングは適切か

#### コード品質
- [ ] DRY原則に従っているか
- [ ] 関数・変数名は明確か
- [ ] 複雑すぎる処理はないか
- [ ] 不要なコードはないか

#### セキュリティ
- [ ] ユーザー入力のバリデーション
- [ ] XSS/CSRF対策
- [ ] 機密情報の漏洩リスク

#### パフォーマンス
- [ ] 不要な再レンダリング
- [ ] N+1クエリ
- [ ] メモリリーク

#### テスト
- [ ] テストは追加されているか
- [ ] カバレッジは十分か

#### ドキュメント
- [ ] 必要なコメントがあるか
- [ ] 破壊的変更はドキュメント化されているか

### 3. 出力形式

```markdown
## PR概要
[PRの目的と変更内容の要約]

## 良い点
- [良い実装や工夫点]

## 要修正
### [重要度: Critical/High/Medium/Low]
**ファイル**: path/to/file.ts:123
**問題**: 問題の説明
**提案**:
\`\`\`typescript
// 修正案のコード
\`\`\`

## 確認事項
- [レビュアーに確認してほしい点]

## 総評
[全体的な評価とマージ可否の判断]
```
