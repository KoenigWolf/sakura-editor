# Code Review

コードレビューを実行します。

## 手順

1. `git diff main...HEAD` で変更内容を確認
2. 以下の観点でレビュー:
   - **バグ・論理エラー**: 潜在的なバグや論理的な問題
   - **セキュリティ**: XSS、インジェクション等の脆弱性
   - **パフォーマンス**: 非効率な処理やメモリリーク
   - **型安全性**: TypeScriptの型エラーや any の乱用
   - **コードスタイル**: プロジェクトの規約との整合性
   - **i18n**: ハードコードされたテキストがないか

## 出力形式

各問題について以下の形式で報告:

```
### [重要度: 高/中/低] ファイル名:行番号

**問題**: 問題の説明
**提案**: 修正案
```

## 実行

変更がない場合は、ステージングされたファイルまたは最新のコミットをレビューします。
